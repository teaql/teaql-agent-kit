# SQL Trace Notes

The TeaQL environment variables TEAQL_SQL=_full, TEAQL_SQL_TABLES=..., and TEAQL_SINK=_stdout
were validated by teaql-tool-core but did not produce visible SQL output in the
rusqlite-backend application runtime.

This is a known area for investigation: the SQL trace env vars are recognized
by the framework, but the generated code's audit configuration may route output
differently depending on the teaql-data-service backend version.

Available env vars (from teaql-tool-core):
  TEAQL_AUDIT  → _silent | _summary | _full
  TEAQL_SQL    → _silent | _summary | _full
  TEAQL_SQL_TABLES → platform_data, school_type_data, school_data
  TEAQL_SINK   → _stdout | _file | _both
  TEAQL_SCHEMA → execute | dry_run | verify

This will be documented for future evaluation reports to ensure SQL trace
capture is reliably demonstrated.
