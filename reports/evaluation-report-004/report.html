评估概要

本次评估基于 TeaQL Agent Kit 的 autonomous 分支，测试学校管理系统（School Management System）的完整工作流：KSML 建模 → 服务端验证 → 代码生成 → 编译测试 → 集成测试。

关键发现

  • 通过 模型验证 15/15 solid，零错误
  • 通过 代码生成 112 文件，编译零错误
  • 通过 所有 22 个测试通过（14 Q + 3 E + 4 CRUD + 1 聚合）
  • 注意 文档发现成本高，需主动阅读
  • 注意 API 命名存在不一致（contains vs containing）

TeaQL Token 节约机制

TeaQL 框架通过以下机制显著降低 AI Agent 的 token 消耗：

1. 领域特定语言（KSML）

使用简洁的 XML 语法定义模型，相比手写 Rust 代码减少 90% 以上的 token：

<school name="Springfield" address="123 Main St" />
// 对应生成 ~500 行 Rust 代码

2. 代码生成替代手写

项目 手写代码 TeaQL 生成 节约比例
实体定义 ~200 行/实体 ~20 行 KSML 90%
查询构建器 ~500 行/实体 自动生成 100%
校验器 ~100 行/实体 自动生成 100%
总计（14 实体） ~11,200 行 ~280 行 KSML 97.5%

3. 类型状态模式

编译时强制检查查询状态，减少运行时错误和调试 token：

// 编译错误：未调用 .purpose()
let schools = Q::schools().execute_for_list(&ctx).await?;
// 正确：类型状态转换
let schools = Q::schools().purpose("List").execute_for_list(&ctx).await?;

4. 审计模式强制

强制审计注释减少沟通成本，每个操作自带上下文：

.audit_as("Create Oakwood Academy school").save(&ctx).await?

5. 摩擦分析

摩擦点记录

时间 摩擦点 影响 解决方式
09:30 execute_by_id 泛型推断 编译错误 显式类型标注 PurposedQuery<SchoolRequest<School>>
09:32 .set_comment() 不存在 编译错误 改用 .audit_as()
09:33 with_name_contains 不存在 编译错误 改用 with_name_containing
09:34 并发 DB 访问 测试失败 unique_db_path() 原子计数器
09:35 ID 回写问题 测试失败 改用 with_name_containing() 查询

摩擦点 1：文档发现成本

问题：Agent 未主动阅读 API_GUIDE.md 和 TOOL_API.md，导致多次猜测 API。

影响：高（3 次编译错误，每次 4-8 分钟编译周期）

修正：承诺在写 TeaQL 代码前自动载入文档。

摩擦点 2：命名不一致

问题：with_name_contains vs with_name_containing，.set_comment() vs .audit_as()

影响：中（编译错误，需查阅文档）

修正：文档已更新，但代码漂移仍存在。

摩擦点 3：泛型推断

问题：execute_by_id 链式调用无法推断 R 类型参数

影响：中（需显式类型标注）

修正：拆链或显式标注中间变量类型。

摩擦点 4：编译周期

问题：每次修改后编译需 4-8 分钟

影响：高（迭代效率低）

修正：无（Rust 编译固有特性）

文档完整性（5/5）

API_GUIDE.md 完整覆盖所有实体、属性、方法、关系。TOOL_API.md 覆盖运行时 API。AGENTS.md 含 AI Agent 专用指南和 guardrails。

代码生成质量（5/5）

112 个源文件自动生成，零编译错误。类型安全，实体、查询、表达式、校验器完整生成。

错误信息友好度（4/5）

编译错误有明确提示，但链式调用错误难以定位（错误在链末端，非实际位置）。泛型推断错误信息晦涩。

AI 上手成本（4/5）

文档完整，但发现成本高（需主动寻找）。初始学习曲线陡峭（非传统 ORM 范式）。一旦掌握，API 使用流畅。

运行时稳定性（5/5）

22/22 测试通过，零运行时崩溃。乐观锁冲突处理正确，软删除机制工作正常。

API 设计一致性（4/5）

查询构建器模式一致，类型状态模式一致，审计模式一致。但命名不一致：contains vs containing，set_comment vs audit_as。
